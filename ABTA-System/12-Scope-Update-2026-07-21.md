## 2) Broadcast (ส่งข้อความแบบกลุ่ม) — ยังอยู่ Phase 3–4 · ⏸️ เลื่อนทำหลัง Phase 1

> **ไม่ย้ายเข้า Phase 1** ในเอกสารสัญญา/งบ  
> **ไม่ทำคู่ขนาน** ในรอบปิดช่องว่าง Phase 1  
> พอ Phase 1 เสร็จค่อยเริ่ม — เก็บเงินตามราคาเฟสนั้น / ตามที่ยืนยันกับลูกค้า

### คืออะไร

เจ้าหน้าที่พิมพ์/เลือกข้อความใน **Back Office** → ส่งผ่าน LINE → สมาชิกที่เลือก (ผูก LINE แล้ว) ได้รับใน LINE OA  
= **Broadcast LINE OA** ในแผนเดิม

### ขอบเขตตามต๋อย (10:33, 14:15, 14:21) + ความเข้าใจทีม

1. เมนู **ส่งข้อความแบบกลุ่ม** ใน Back Office  
2. กรองด้วย **ประเภทสมาชิก** และ **สถานภาพ** (ทีมตีความคำว่า Tag ในแชท = สองฟิลด์นี้ ไม่แยก Tag อิสระใหม่ — ถ้ายืนยันผิดค่อยปรับ)  
3. กดเลือกสมาชิก → ส่ง  
4. ใช้ครอบคลุมการส่งข้อความหลายสถานการณ์ (เชิญประชุม ฯลฯ)

### แจ้งเตือนกรรมการกลางมี.ค. — แนวทีม (รอยืนยันในแชท)

ไม่ทำเป็น cron อัตโนมัติอย่างเดียวในรอบแรก  
แนวทาง: ใน Back Office มีงาน/วันที่ + รายละเอียด แล้ว**กดแจ้งเตือนเอง**ไปยังกลุ่ม **กรรมการสมาคม** (เพื่อเตือนต่ออายุก่อนประชุมใหญ่เม.ย.)

## 3) คำถามที่ยังจะถามลูกค้า / สิ่งที่เคลียร์แล้ว

### เคลียร์แล้วฝั่งทีม

| เรื่อง | มติ |
|--------|-----|
| Excel หัวเปล่า | อิง `NewMemDatabase.xlsx` · มีไฟล์หัวอย่างเดียวที่ `Data/NewMemDatabase-blank-headers.xlsx` |
| Tag | = ประเภทสมาชิก + สถานภาพ |
| แจ้งเตือนกรรมการกลางมี.ค. | กดส่งเองจากหลังบ้าน (งาน/วันที่ + รายละเอียด → กลุ่มกรรมการ) |

### ยังถามลูกค้า (ในข้อความตอบ)

1. รายงาน/คำสั่งตรวจสมาชิก + ภาพกรรมการ 11–15 คน — ขอตัวอย่าง/ความหมาย  
2. ข้อความเชิญประชุมแก้ในชีตแล้วกดส่ง — ยังต้องมีแม่แบบในระบบไหม หรือใช้หน้าส่งข้อความกลุ่มพิมพ์/วางข้อความแล้วส่งพอ

### ยังไม่ทำ

Export Excel · Self-Service · PDF เต็ม · OCR · QR · Member History · Backup
