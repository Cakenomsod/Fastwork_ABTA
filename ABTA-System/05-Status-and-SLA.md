# สถานะในระบบ และ SLA

---

## สถานะสมาชิก (Member Status)

```
สมัครแล้ว
    ↓
รอตรวจสอบเอกสาร
    ↓
สมาชิกชั่วคราว  ← ได้ Member ID ชั่วคราวทันทีหลังสมัคร
    ↓
สมาชิกสมบูรณ์  ← หลังนายทะเบียนอนุมัติ + ออกเลขถาวร
    ↓
ใกล้หมดอายุ  ← ระบบแจ้งเตือนอัตโนมัติ
    ↓
หมดอายุ
```

| สถานะ | ความหมาย | Phase |
|-------|----------|-------|
| สมัครแล้ว | กรอกฟอร์มแล้ว รอดำเนินการ | Phase 1 |
| รอตรวจสอบเอกสาร | ส่งเอกสารแล้ว รอเจ้าหน้าที่ | Phase 1 |
| สมาชิกชั่วคราว | ได้ Member ID ชั่วคราว ใช้สิทธิ์ได้ | Phase 1 |
| สมาชิกสมบูรณ์ | อนุมัติแล้ว มีเลขถาวร | Phase 1 |
| ใกล้หมดอายุ | ใกล้ครบกำหนดต่ออายุ | Phase 1 |
| หมดอายุ | หมดอายุแล้ว ต้องต่ออายุ | Phase 1 |

## สถานะการชำระเงิน (Payment Status)

```
รอชำระเงิน
    ↓
ได้รับหลักฐานการชำระเงินแล้ว
    ↓
ออกใบเสร็จชั่วคราวแล้ว  ← ทันทีหลังสมัคร + แนบสลิป
    ↓
รอตรวจสอบข้อมูล (แอดมิน/นายทะเบียน)
    ↓
รอตรวจสอบสลิป (เหรัญญิก)
    ↓
ยืนยันการชำระเงินแล้ว
    ↓
ออกใบเสร็จตัวจริงแล้ว
```

| สถานะ | ความหมาย | Phase |
|-------|----------|-------|
| รอชำระเงิน | ยังไม่ได้รับสลิป | Phase 1 |
| ได้รับหลักฐานการชำระเงินแล้ว | ระบบรับสลิปแล้ว | Phase 1 |
| ออกใบเสร็จชั่วคราวแล้ว | มีเลขใบเสร็จชั่วคราว | Phase 1 |
| รอตรวจสอบข้อมูล | แอดมิน/นายทะเบียนกำลังตรวจ | Phase 1 |
| รอตรวจสอบสลิป | เหรัญญิกกำลังตรวจ | Phase 1 |
| ยืนยันการชำระเงินแล้ว | เหรัญญิกอนุมัติสลิปแล้ว | Phase 1 |
| ออกใบเสร็จตัวจริงแล้ว | เปลี่ยนจากชั่วคราวเป็นตัวจริง | Phase 1 |
| ปฏิเสธ — ออกเลขใบเสร็จใหม่ | สลิปไม่ผ่าน ต้องส่งใหม่ | Phase 1 |

## สถานะใบเสร็จ (Receipt Status)

```
ชั่วคราว  ← ออกทันทีหลังสมัคร
    ↓
รอตรวจสอบ
    ↓
ตัวจริง  ← เหรัญญิกอนุมัติสลิป
    ↓
(ถ้าไม่ผ่าน) ยกเลิก → ออกเลขใบเสร็จใหม่
```

| สถานะ | ความหมาย | Phase |
|-------|----------|-------|
| `temp` | ใบเสร็จชั่วคราว — ออกทันทีหลังสมัคร | Phase 1 |
| `pending_review` | รอเหรัญญิกตรวจสลิป | Phase 1 |
| `official` | ใบเสร็จตัวจริง — เหรัญญิกอนุมัติแล้ว | Phase 1 |
| `rejected` | ไม่ผ่าน — ออกเลขใบเสร็จใหม่ | Phase 1 |

## สถานะสัมมนา (Seminar Status)

```
ลงทะเบียนแล้ว
    ↓
ชำระเงินแล้ว
    ↓
ยืนยันสิทธิ์แล้ว
    ↓
ออก QR Check-in แล้ว    ← Phase 3–4
    ↓
เข้าร่วมงานแล้ว         ← Phase 3–4
```

| สถานะ | Phase |
|-------|-------|
| ลงทะเบียนแล้ว | Phase 1 |
| ชำระเงินแล้ว | Phase 1 |
| ยืนยันสิทธิ์แล้ว | Phase 1 |
| ออก QR Check-in แล้ว | Phase 3–4 |
| เข้าร่วมงานแล้ว | Phase 3–4 |

## SLA มาตรฐาน ABTA

| กระบวนการ | เป้าหมาย | ผู้รับผิดชอบ |
|-----------|----------|-------------|
| สมัครสมาชิก → ได้ Member ID ชั่วคราว | **ทันที** (ภายใน 5 นาที) | ระบบ (อัตโนมัติ) |
| ระบบยืนยันรับสลิป | ภายใน **24 ชั่วโมง** | ระบบ + เจ้าหน้าที่ |
| ตรวจสอบข้อมูลสมาชิก (ขั้นที่ 1) | ภายใน **7 วันทำการ** | นายทะเบียน |
| ตรวจสอบสลิป + ออกใบเสร็จตัวจริง (ขั้นที่ 2) | ภายใน **7 วันทำการ** | เหรัญญิก |
| ออกเลขสมาชิกถาวร | ภายใน **7 วันทำการ** | เหรัญญิก (หลังอนุมัติสลิป) |
| ออกใบเสร็จชั่วคราว | **ทันที** (ภายใน 5 นาที) | ระบบ (อัตโนมัติ) |

## ข้อมูลที่แสดงเมื่อเช็คสถานะ (LINE OA)

สมาชิกพิมพ์ **"เช็คสถานะ"** จะเห็น:

| ข้อมูล | แหล่ง |
|--------|-------|
| Member ID | ฐานข้อมูลสมาชิก |
| บัตรสมาชิก (Member ID Card) | `memberCardUrl` |
| ใบเสร็จ (ชั่วคราว / ตัวจริง) | `receiptUrl` + Receipt Status |
| สถานะสมาชิก | Member Status |
| สถานะการชำระเงิน | Payment Status |
| สถานะการสมัครสัมมนา | Seminar Status |
| วันหมดอายุสมาชิก | ฐานข้อมูลสมาชิก |
| สถานะการต่ออายุ | Renewal Status |

> ใน Phase 2 จะเพิ่ม: หน้าเว็บ Self-Service (Login + Dashboard + ดาวน์โหลดเอกสารเต็มรูปแบบ)
