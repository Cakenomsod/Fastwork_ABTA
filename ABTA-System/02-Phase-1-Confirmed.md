## ยืนยัน Flow Phase 1 เพิ่มเติม (11 ก.ค. 2569)

จากการสรุป Flow กับลูกค้า — ยืนยันว่าอยู่ใน Phase 1 ดังนี้:

| หัวข้อ | การตัดสินใจ Phase 1 |
|--------|---------------------|
| **แยกขั้นตรวจ 2 คน** | แอดมิน/นายทะเบียนตรวจ **ข้อมูลสมาชิก** ก่อน → เหรัญญิกตรวจ **สลิป/ใบเสร็จ** ทีหลัง |
| **ใบเสร็จชั่วคราว → จริง** | ออกใบเสร็จชั่วคราว**หลังนายทะเบียนอนุมัติข้อมูล** → เหรัญญิกผ่านแล้วเปลี่ยนเป็นตัวจริง / ไม่ผ่านออกเลขใบเสร็จใหม่ |
| **เลขสมาชิก** | ออก Member ID ชั่วคราว**ทันทีหลังสมัคร** → ผ่านนายทะเบียน promote เป็นถาวร + สมาชิกสมบูรณ์ / ไม่ผ่าน → ออกเลขสมาชิกชั่วคราวใหม่ |
| **แสดงผลให้สมาชิก** | ผ่าน LINE OA แสดง **บัตรสมาชิก (ID Card)** + **ใบเสร็จ** (ไม่ใช่แค่ข้อความสถานะ) |
| **สัมมนา 3 แบบ** | คนทั่วไปเสียเงิน / สมาชิกฟรี / สมาชิกเสียเงิน — กำหนดได้ต่องาน |
| **ช่องทาง** | ฝั่งสมาชิกทั้งหมดผ่าน **LINE OA** (ฟอร์มเปิดเป็น Web/LIFF จาก Rich Menu) — **Back Office แยก** สำหรับเจ้าหน้าที่ |
| **LINE channels** | **Messaging API** (บอท/แจ้งเตือน) + **LINE Login** (LIFF ฟอร์ม) — 2 channel ภายใต้ Provider เดียวกัน ([รายละเอียด](./07-Tech-Stack.md)) |
| **สมาชิกเก่ายังไม่ผูก LINE** | กรอกข้อมูลยืนยันตัวตน → ค้นหาในฐานข้อมูลเก่า → ยืนยันผูก LINE UID → ออกเลขสมาชิกรูปแบบใหม่ |
| **สิทธิ์สมาชิกชั่วคราว** | ใช้สิทธิ์ครบเหมือนสมาชิกสมบูรณ์ — "ชั่วคราว" = รอตรวจสอบเลข/ใบเสร็จ ไม่ใช่จำกัดสิทธิ์ |
| **ปฏิเสธข้อมูล (ขั้นที่ 1)** | ไม่ส่งต่อเหรัญญิก → แจ้งสมาชิกแก้ไข/ส่งใหม่ → วนกลับนายทะเบียน |
| **ปฏิเสธสลิป (ขั้นที่ 2)** | สมาชิกสมบูรณ์แล้ว → ออกเลขใบเสร็จใหม่ → สมาชิกส่งสลิปใหม่ |
| **Edge cases พิเศษ** | จัดการผ่านข้อความในระบบ + แชท LINE กลุ่ม / ติดต่อแอดมิน |

> **หมายเหตุ:** การสร้าง PDF อัตโนมัติแบบเต็มรูปแบบ (template สวยงาม + export หลายแบบ) ยังวางไว้ **Phase 2** — Phase 1 เน้น workflow เลขที่/สถานะ + แสดงเอกสารใน LINE OA

## 6 ระบบใน Phase 1

### 1. ระบบรับสมัครสมาชิก

**Workflow ที่ตกลง (อัปเดต 13 ก.ค. 2569):**

```
เปิดจาก LINE OA
    ↓
เช็ค LINE User ID — ผูกแล้ว → ใช้งานปกติ (สมัคร/ต่ออายุ/เช็คสถานะ)
ยังไม่ผูก
    ↓
┌─ สมาชิกใหม่ (ไม่เคยมีในฐานข้อมูล) ─────────────────────────┐
│  กรอกข้อมูล → แนบสลิป → ออก Member ID ชั่วคราว → Back Office │
└──────────────────────────────────────────────────────────────┘
┌─ สมาชิกเก่า (มีในฐานข้อมูลเก่า แต่ยังไม่ผูก LINE) ──────────┐
│  กรอกข้อมูลยืนยันตัวตน → ค้นหาเลขสมาชิกเก่า → ยืนยันผูก LINE │
│  → ออกเลขสมาชิกรูปแบบใหม่ → เข้าใช้ระบบได้                    │
└──────────────────────────────────────────────────────────────┘
```

**Flow สมัครสมาชิกใหม่:**

```
เปิดจาก LINE OA (ยังไม่ผูก LINE + เลือกสมัครใหม่)
    ↓
กรอกข้อมูลส่วนตัว
    ↓
แนบสลิปโอนเงิน (ขั้นตอนเดียว)
    ↓
บันทึกลงฐานข้อมูล
    ↓
ออก Member ID ชั่วคราว ทันที (ภายใน 5 นาที)
    ↓
แจ้งยืนยันผ่าน LINE OA (+ เลขสมาชิก)
    ↓
Back Office ขั้นที่ 1: แอดมิน/นายทะเบียนตรวจข้อมูล
    ↓ (ผ่าน → promote เลขถาวร + ออกใบเสร็จชั่วคราว / ไม่ผ่าน → ออกเลขสมาชิกใหม่)
Back Office ขั้นที่ 2: เหรัญญิกตรวจสลิป
    ↓ (ผ่าน → ใบเสร็จตัวจริง / ไม่ผ่าน → ออกเลขใบเสร็จใหม่)
```

**ข้อมูลที่เก็บ:**

- Member ID (ชั่วคราว → ถาวร / รูปแบบใหม่หลังผูก LINE สมาชิกเก่า)
- เลขที่ใบเสร็จ (ชั่วคราว → ตัวจริง)
- ชื่อ / นามสกุล
- ชื่อนิติบุคคล
- เบอร์โทรศัพท์
- อีเมล
- หน่วยงาน / ชื่อตึก
- LINE User ID
- เลขสมาชิกเก่า (legacy) — อ้างอิงจากฐานข้อมูลเดิม
- สถานะสมาชิก
- วันหมดอายุสมาชิก
- URL บัตรสมาชิก (Member ID Card)

**ฟีเจอร์เพิ่มเติม:**

- **ผูก LINE UID สมาชิกเก่า** — ยืนยันตัวตนก่อนผูก (ดูรายละเอียดด้านล่าง)
- ดึงข้อมูลสมาชิกจาก LINE User ID ที่ผูกแล้ว (ไม่ต้องกรอกซ้ำ)
- ต่ออายุสมาชิกผ่าน LINE OA
- แจ้งเตือนก่อนหมดอายุอัตโนมัติ **45 วัน** และ **15 วัน** ก่อนวันหมดอายุ (สมาชิกสามัญ/วิสามัญหมดอายุ **31 ธ.ค. ทุกปี** — อัปเดต 21 ก.ค. 2569)

#### ผูก LINE UID สมาชิกเก่า (ยืนยันตัวตน)

> สมาชิกที่มีอยู่ในฐานข้อมูลเก่า แต่ยังไม่เคยผูก LINE User ID ต้องยืนยันตัวตนก่อนเข้าใช้ระบบใหม่

```
เปิดจาก LINE OA (ยังไม่ผูก LINE)
    ↓
เลือก "ยืนยันสมาชิกเก่า" / ระบบนำทางอัตโนมัติ
    ↓
กรอกข้อมูลยืนยันตัวตน:
  ชื่อ, นามสกุล, ชื่อนิติบุคคล, ชื่อตึก, อีเมล, เบอร์โทร
    ↓
ระบบค้นหาในฐานข้อมูลสมาชิกเก่า (match หลายฟิลด์ร่วมกัน)
    ↓
พบ 1 รายการ → แสดงเลขสมาชิกเก่าให้ยืนยัน
พบหลายรายการ → ให้เลือกรายการที่ตรง (แสดงชื่อนิติบุคคล/ตึกเพื่อแยก)
ไม่พบ → นำไปสมัครสมาชิกใหม่
    ↓
ยืนยันผูก LINE UID
    ↓
ออกเลขสมาชิกรูปแบบใหม่ (เช่น ABTA-2026-XXXX) + เก็บ legacyMemberId อ้างอิง
    ↓
เข้าใช้ระบบใหม่ได้ (เช็คสถานะ, ต่ออายุ, สัมมนา ฯลฯ)
```

**ข้อควรระวัง — ข้อมูลซ้ำในฐานข้อมูลเก่า:**

- อีเมลหรือเบอร์โทรเดียวกันอาจผูกกับสมาชิกหลายคน (แอดมินยุคก่อนใช้อีเมลตัวเองสมัครแทนสมาชิกที่ไม่มีอีเมล)
- **ห้ามค้นหาด้วยอีเมลหรือเบอร์อย่างเดียว** — ต้องใช้ชื่อ + นามสกุล + ชื่อนิติบุคคล + ชื่อตึก ร่วมกันเพื่อแยกรายการ
- ถ้ายังพบหลายรายการ → แสดงรายการให้ผู้ใช้เลือกรายการที่ตรงก่อนยืนยัน

> สมาชิกเก่าที่ผูก LINE สำเร็จแล้ว **ไม่ต้องชำระค่าสมัครใหม่** — เป็นการยืนยันตัวตนและผูกบัญชีเท่านั้น (ต่างจากสมัครสมาชิกใหม่ที่ต้องแนบสลิป)

### 2. ระบบรับสมัครสัมมนา

- ลงทะเบียนเข้าร่วมสัมมนา
- ดึงข้อมูลสมาชิกเดิมอัตโนมัติ
- **รองรับ 3 ประเภทราคา:** คนทั่วไปจ่าย / สมาชิกฟรี / สมาชิกจ่าย
- รองรับข้อมูลเพิ่มเติม: ไซส์เสื้อ, ประเภทอาหาร, ข้อมูลพิเศษ
- แสดงสถานะการสมัคร
- แสดงค่าลงทะเบียนตามประเภทผู้สมัคร
- อัปโหลดสลิปเมื่อต้องชำระเงิน

### 3. ระบบอัปโหลดหลักฐานการชำระเงิน (Manual)

> ⚠️ Phase 1 **ไม่มี** OCR หรือ AI ตรวจสลิปอัตโนมัติ

**ฝั่งสมาชิก:**
- อัปโหลดสลิป (รวมในขั้นตอนสมัคร หรือส่งผ่าน LINE OA)
- ได้รับ Member ID ชั่วคราวทันทีหลังส่ง — ใบเสร็จชั่วคราวออก**หลังนายทะเบียนอนุมัติข้อมูล**
- ระบบบันทึกและแจ้งเตือนเจ้าหน้าที่

**ฝั่งเจ้าหน้าที่ (นายทะเบียน — ขั้นที่ 1):**
- ตรวจสอบข้อมูลสมาชิก
- อนุมัติแล้ว → **promote เลขถาวร** + **ออกใบเสร็จชั่วคราว** + ส่งต่อเหรัญญิก
- ไม่ผ่าน → แจ้งสมาชิกแก้ไข/ส่งใหม่ — **ไม่ส่งต่อเหรัญญิก** จนกว่าข้อมูลจะผ่าน

**ฝั่งเจ้าหน้าที่ (เหรัญญิก — ขั้นที่ 2):**
- เปลี่ยนสถานะใบเสร็จ: ชั่วคราว → ตัวจริง
- ไม่ผ่าน → ออกเลขใบเสร็จใหม่ + แจ้งสมาชิกส่งสลิปใหม่ (สมาชิกยังเป็นสมาชิกสมบูรณ์)

| ตรวจสอบสลิปและใบเสร็จ | ยืนยันการชำระ + ออกใบเสร็จตัวจริง | เหรัญญิก |

| นำเข้า Excel สมาชิกเก่า | ตามรูปแบบไฟล์ที่ลูกค้าเคยส่ง (`NewMemDatabase.xlsx`) | แอดมิน |

### 5. ระบบแจ้งเตือนผ่าน LINE OA

**แจ้งสมาชิก:**
- ผูก LINE UID สมาชิกเก่าสำเร็จ (+ เลขสมาชิกรูปแบบใหม่)
- สมัครสมาชิกสำเร็จ (+ Member ID ชั่วคราว)
- นายทะเบียนอนุมัติข้อมูลแล้ว (+ เลขถาวร + ใบเสร็จชั่วคราว)
- ส่งหลักฐานการชำระเงินแล้ว
- เหรัญญิกอนุมัติสลิปแล้ว (+ ใบเสร็จตัวจริง)
- สถานะสมาชิกเปลี่ยน / ต้องแก้ไขข้อมูล (ขั้นที่ 1)
- สลิปไม่ผ่าน / ต้องส่งสลิปใหม่ (ขั้นที่ 2)
- แจ้งเตือนก่อนหมดอายุอัตโนมัติที่ **45 วัน** และ **15 วัน** ก่อนวันหมดอายุ  
  (กฎธุรกิจจากลูกค้า: สมาชิกสามัญ/วิสามัญ **หมดอายุ 31 ธันวาคม ของทุกปี**)

**แจ้งเจ้าหน้าที่:**
- มีสมาชิกใหม่
- มีการส่งหลักฐานการชำระเงิน
- มีรายการรออนุมัติ (แยกคิวข้อมูล vs การเงิน)

### 6. ระบบตรวจสอบสถานะผ่าน LINE OA ⭐ ลำดับแรก

> ลูกค้าขอให้พัฒนาส่วนนี้ **ก่อนเป็นอันดับแรก** (ประมาณ 5–6 วัน)

สมาชิกพิมพ์คำสั่ง เช่น **"เช็คสถานะ"** แล้วบอทตอบกลับอัตโนมัติ:

- Member ID + สถานะสมาชิก
- **บัตรสมาชิก (Member ID Card)**
- **ใบเสร็จ** (ชั่วคราว / ตัวจริง)
- สถานะการชำระเงิน
- สถานะการสมัครสัมมนา
- วันหมดอายุสมาชิก
- สถานะการต่ออายุสมาชิก

**ทางเลือกที่ตกลง:**

## เป้าหมายเมื่อ Phase 1 เสร็จ

**สมาชิกสามารถ:**
- ยืนยันตัวตนสมาชิกเก่าและผูก LINE UID เพื่อเข้าใช้ระบบใหม่
- สมัครสมาชิก / ต่ออายุ / สัมมนา ผ่าน LINE OA
- ส่งหลักฐานการชำระเงิน
- ได้รับ Member ID ชั่วคราวทันทีหลังสมัคร
- ใช้สิทธิ์สมาชิกครบตั้งแต่ได้เลขชั่วคราว (รอตรวจสอบเลข/ใบเสร็จเท่านั้น)
- ได้รับเลขถาวร + สถานะสมาชิกสมบูรณ์หลังนายทะเบียนอนุมัติข้อมูล
- ได้รับใบเสร็จชั่วคราวหลังนายทะเบียนอนุมัติข้อมูล
- เช็คสถานะ + ดูบัตรสมาชิก + ใบเสร็จ ผ่าน LINE OA
- ได้รับแจ้งเตือนก่อนหมดอายุและต่ออายุได้ทันเวลา

**เจ้าหน้าที่สามารถ:**
- แยกขั้นตรวจข้อมูล (นายทะเบียน) และการเงิน (เหรัญญิก)
- จัดการเลขสมาชิกและเลขใบเสร็จเมื่อไม่ผ่านการตรวจ
- อนุมัติสมาชิกและสัมมนาผ่านระบบหลังบ้าน
- ลดการตอบคำถามซ้ำเรื่องสถานะต่าง ๆ
